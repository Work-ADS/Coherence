Foundations

Accessibility

Accessibility is in the markup from the first commit — not a follow-up task. Every component ships with ARIA, keyboard support, and contrast compliance. If it doesn't pass these checks, it doesn't ship.

Color contrast

WCAG 2.1 AA is the baseline. Our action-500 achieves ~6.3:1 against white. All semantic color pairings are pre-validated — if you use tokens correctly, contrast is guaranteed.

Ratio Applies to Level

}

Rules

Do
  • Use Angular CDK for focus trap, a11y, overlay
  • Test with keyboard-only navigation before shipping
  • Use semantic HTML (button, nav, main, aside) first
Don't
  • Never use outline: none without a replacement indicator
  • Never rely on color alone to communicate state
  • Never use tabindex > 0 — fix DOM order instead
