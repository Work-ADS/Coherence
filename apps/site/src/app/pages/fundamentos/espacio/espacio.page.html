Foundations

Space

Everything sits on a 4px grid. The dimension scale is the primitive — --dimension-N = N × 4px. Semantic spacing tokens give those numbers a role so you never have to think about pixel values.

Dimension scale

137 tokens from 0 to 800px. Below are the ones you'll use 90% of the time. The rule: components never use these directly — they consume semantic tokens like --space-md or --button-height.

@for (d of baseScale; track d.index) {
--dimension-{{ d.index }} {{ d.px }}px
}

Semantic spacing

Named by T-shirt size. Each maps to a dimension primitive and carries a usage role. Use these in components — never raw dimension tokens.

@for (s of semanticSpacing; track s.name) {
{{ s.name }} {{ s.maps }} {{ s.use }}
}

Border radius

Radius lives in the dimension file because it's a spatial value — but it's not on the 4px grid (radius-full = 9999px for pills). Kept separate from the main scale so the rule is clear.

@for (r of radiusScale; track r.name) {
{{ r.name }} {{ r.value }} {{ r.use }}
}

Rules

Do
  • Use semantic spacing: padding: var(--space-md)
  • Use semantic radius: border-radius: var(--radius-md)
  • Let responsive tokens handle breakpoint scaling
Don't
  • Never padding: 16px — use the token
  • Never reference --dimension-4 in a component — use --space-md
  • Never invent radius values outside the scale
