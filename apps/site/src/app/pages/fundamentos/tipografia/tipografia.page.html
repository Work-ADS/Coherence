Foundations

Typography

One family everywhere: Roboto Serif. Hierarchy comes from weight, size, and tracking — not from a second typeface. The register is editorial-software: Medium pre-redesign, NYT web, Bloomberg Terminal.

Heading scale

Responsive — mobile-first values scale up at the lg breakpoint (992px). Uses the font shorthand: weight size/line-height family.

@for (t of headingScale; track t.name) {
{{ t.sample }}
{{ t.token }} {{ t.note }}
}

Body scale

Fixed across breakpoints — readability at 12–16px doesn't benefit from scaling. Token naming: --type-body-[size]-[weight].

@for (t of bodyScale; track t.name) {
The quick brown fox jumps
{{ t.token }} {{ t.spec }} {{ t.note }}
}

Letter spacing

Five tracking tokens. Tight for large headings, wide for uppercase utility text.

@for (t of trackingScale; track t.name) {
COHERENCE DESIGN
{{ t.token }} {{ t.value }} {{ t.use }}
}

Rules

Do
  • Use font: var(--type-*) shorthand — sets size, weight, line-height, and family in one declaration
  • Deliver hierarchy through weight + size + tracking only
  • Use tabular figures for data-heavy contexts
Don't
  • Never add a second font family (sans is deferred to v2 brand modes)
  • Never use font-size or font-weight individually — use the shorthand token
  • Never set raw pixel sizes outside libs/tokens/
